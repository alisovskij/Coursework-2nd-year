# WebCrawler (WinForms)

Десктоп-приложение на C# (.NET 8, Windows Forms), которое обходит сайт методом поиска в ширину (BFS), собирает метаданные страниц и показывает результаты в реальном времени, а также сохраняет их в текстовый файл.

## Возможности

- Графический интерфейс на Windows Forms
- Обход сайта по алгоритму BFS с ограничением глубины и количества страниц
- Живое отображение обхода в таблице (`DataGridView`) — строки появляются по мере загрузки
- Кнопка **Стоп** с корректной отменой через `CancellationToken`
- Лог событий и индикатор прогресса
- Фильтрация ссылок по домену и расширению файла
- Настраиваемая задержка между запросами
- Сохранение результатов и статистики в текстовый файл

## ⚠️ Платформа

Приложение использует **Windows Forms**, поэтому **собирается и запускается только на Windows**. На Linux/macOS его можно лишь скомпилировать (без запуска) с флагом `EnableWindowsTargeting`.

## Требования

- Windows 10/11
- [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)
- (опционально) Visual Studio 2022 с компонентом «.NET desktop development»

## Запуск на Windows

### Через командную строку

```bash
cd Koleso
dotnet run
```

### Через Visual Studio

1. Открыть `WebCrawler.csproj`
2. Нажать **F5** (запуск с отладкой) или **Ctrl+F5** (без отладки)

### Сборка релизной версии

```bash
dotnet build -c Release
./bin/Release/net8.0-windows/WebCrawler.exe
```

## Как пользоваться

1. Введите **URL** стартовой страницы (схему `https://` можно опустить — она добавится автоматически).
2. Настройте параметры:
   - **Глубина** — насколько глубоко уходить от стартовой страницы
   - **Макс. стр.** — лимит обработанных страниц
   - **Задержка(мс)** — пауза между запросами
   - **Оставаться на домене** — не переходить на чужие домены
   - **Файл вывода** — куда сохранить отчёт (кнопка «Обзор…» открывает диалог)
3. Нажмите **▶ Старт**. Строки будут появляться в таблице по мере обхода; страницы с ошибками подсвечиваются розовым.
4. В любой момент можно нажать **■ Стоп** для остановки.
5. По завершении отчёт сохраняется в указанный файл.

## Архитектура

Код разбит на файлы по ответственности — логика обхода полностью отделена от интерфейса:

```
Koleso/
├── Models.cs        # PageInfo, CrawlerConfig, CrawlUpdate — модели данных
├── Crawler.cs       # Crawler — логика обхода (не зависит от UI)
├── MainForm.cs      # MainForm — окно Windows Forms
├── Program.cs       # точка входа приложения
├── WebCrawler.csproj
└── README.md
```

### Разделение логики и UI

`Crawler` ничего не знает о Windows Forms. Связь с интерфейсом — через стандартные механизмы:

- **`IProgress<CrawlUpdate>`** — краулер сообщает о каждой обойдённой странице;
- **`IProgress<string>`** — текстовые статусные сообщения для лога;
- **`CancellationToken`** — отмена обхода по кнопке «Стоп».

`Progress<T>` автоматически перенаправляет вызовы в UI-поток, поэтому форма безопасно обновляет элементы без ручного `Invoke`.

```
MainForm (UI-поток)                 Crawler (фоновая задача)
    │                                      │
    │  CrawlAsync(url, progress, ct) ─────▶│
    │                                      │  загрузка страницы
    │◀──── IProgress.Report(update) ───────│  (по каждой странице)
    │  AddRow() в DataGridView             │
    │                                      │
    │  кнопка Стоп → cts.Cancel() ────────▶│  ct.ThrowIfCancellationRequested()
```

### Алгоритм обхода (BFS)

```
Стартовый URL (глубина 0)
        │
        ▼
  Загрузить страницу → собрать ссылки → в очередь (глубина 1)
        │
        ▼
  ...до достижения MaxDepth или MaxPages
```

Используется очередь пар `(url, глубина)` и `HashSet` посещённых URL для защиты от повторов.

### Модели данных (`Models.cs`)

| Класс | Назначение |
|---|---|
| `PageInfo` | Метаданные страницы: URL, заголовок, HTTP-статус, размер, число ссылок, глубина, ошибка |
| `CrawlerConfig` | Настройки: глубина, лимит страниц, задержка, домен, файл, расширения |
| `CrawlUpdate` | Одно обновление прогресса (страница + текущий счётчик) |

### Ключевые методы `Crawler` (`Crawler.cs`)

| Метод | Описание |
|---|---|
| `CrawlAsync(...)` | Основной цикл BFS. Принимает каналы прогресса и токен отмены |
| `FetchPageAsync(...)` | **Один** HTTP-запрос: возвращает метаданные **и** HTML |
| `ExtractLinks(html, url)` | Извлекает ссылки из уже загруженного HTML (без повторного запроса) |
| `BuildReport()` | Формирует текст отчёта со статистикой |
| `SaveResults()` | Записывает отчёт в файл |
| `Dispose()` | Освобождает `HttpClient` (реализован `IDisposable`) |

## Что было улучшено по сравнению с консольной версией

- ✅ **Графический интерфейс** вместо консольного ввода
- ✅ **Устранена двойная загрузка** — страница скачивается один раз, HTML переиспользуется
- ✅ **Добавлен `IDisposable`** — `HttpClient` корректно освобождается
- ✅ **Поддержка отмены** через `CancellationToken`
- ✅ **Логика отделена от интерфейса** — `Crawler` можно использовать в любом приложении
- ✅ **Код разбит на файлы** по зонам ответственности

## Ограничения

- Парсинг HTML через регулярные выражения, а не полноценный парсер
- Обход последовательный (без параллельных запросов)
- JavaScript-контент не выполняется (только статический HTML)
- Работает только на Windows (Windows Forms)
